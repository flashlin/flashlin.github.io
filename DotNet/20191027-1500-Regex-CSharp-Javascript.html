# 比較 C# Regex 和 Javascript Regex

表達式說明 C# Javascript
開頭 ^ ^
結尾 $ $
零次或多次, 等於 {0,} * *
一次或多次, 等於 {1,} + +
零次或一次, 等於 {0,1} ? ?
匹配確定的n次 {n} {n}
至少匹配n次 {n,} {n,}
最少匹配n次且最多匹配m次 {n,m} {n,m}
含數字的字串 [0-9] [0-9]
含小寫字母的字串 [a-z] [a-z]
不含英文字母的字串 [a-zA-Z] [^a-zA-Z]
匹配除 "\n" 之外的任何字元, 要匹配包括 "\n" 在內的任何字元, 請使用 "[.\n]" 的範式 . .
匹配 pattern 群組 (pattern) (pattern)
匹配 pattern 但不獲取匹配結果, 不進行儲存供以後使用. 例如：run(?:n|ning) 能匹配 running 整個字串, 並且從此字串後繼續比對 (?:pattern) (?:pattern)
正向預查, 在任何匹配 pattern 的字串開始處匹配尋找字串, 例如 Office(?=2003|2007) 能匹配 Office2003 , 但不能匹配 Office2005, 而從 Office 字串後繼續比對 (?=pattern) (?=pattern)
負向預查, 在任何不匹配 pattern 的字串開始處匹配尋找字串.例如 Office(?!2003|2007) 能匹配 Office2005 , 但不能匹配 Office2010 , 而從 Office 字串後繼續比對 (?!pattern) (?!pattern)
匹配未包含的任意字元 [^xyz] [^xyz]
匹配一個單詞邊界, 也就是指單詞和空格間的位置 \b 不支援
匹配非單詞邊界 \B 不支援
匹配由 x 指明的控制字元. 例如 \cM 匹配一個 Control-M . x 的值必須為 A-Z 或 a-z 之一. 否則將 c 視為一個原義的 "c" 字元 \cx \cx
匹配一個數位字元 \d \d
匹配一個非數位字元 \D \D
匹配一個換頁符 \f \f
匹配一個換行符 \n \n
匹配一個Enter符 \r \r
匹配任何空白字元, 包括空格, 製表符, 換頁符等 \s \s
任何非空白字元(空白,換行,tab) \S \S
匹配一個製表符 \t \t
匹配一個垂直製表符 \v \v
匹配包括底線的任何數字字母, 等於 [A-Za-z0-9_] \w \w
任何非數字字母底線 \W \W
匹配 n , 其中 n 為十六進制轉義值. 十六進制轉義值必須為確定的兩個數位長. 例如 \x41 匹配 A . "\x041" 則等價於"\x04&1" \xn \xn
匹配num , 其中 num 是一個正整數. 對所獲取的匹配的參照. 例如 "(.)\1" 匹配兩個連續的相同字元 \num \num
匹配 n , 其中 n 是一個用四個十六進制數位表示的 Unicode 字元. 例如 "\u00A9" 匹配版權符號 "©" \un \un

Javascript exec 用法, 僅返回找到的第一個匹配項

var re = new RegExp('ab+c');
var myArray = myRe.exec('cdbbdbsbz');
console.log('The value of lastIndex is ' + myRe.lastIndex);

Javascript match 用法, 如果使用 g 正則表達式中的標誌, 則返回所有匹配項

var re = new RegExp('\\w+\\s');
var myArray = myRe.match('fee fi fo fum', 'g');
console.log(myArray);

// ["fee ", "fi ", "fo "]
