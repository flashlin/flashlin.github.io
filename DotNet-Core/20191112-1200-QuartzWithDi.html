# Quartz with Dotnet core DI

如何用Dotnet Core DI 注入Quartz Job 所需的依賴物件(Dependency Objects)?

建立好 Quartz Job 之後

public class SomeJob : IJob 
{
    public SomeJob(...) 
    {
        //...
    }
}

首先自行建立 MyJobFactory 自訂的工廠物件並實作 Quartz 提供的 IJobFactory

public class MyJobFactory : IJobFactory
{
    private readonly IServiceProvider _serviceProvider;

    public MyJobFactory(IServiceProvider serviceProvider)
    {
        _serviceProvider = serviceProvider;
    }

    public IJob NewJob(TriggerFiredBundle bundle, IScheduler scheduler)
    {
        return _serviceProvider.GetService(bundle.JobDetail.JobType) as IJob;
    }

    public void ReturnJob(IJob job)
    {
        var disposable = job as IDisposable;
        disposable?.Dispose();
    }
}

建立自己的排程服務, 並將IJobFactory 從建構元傳入

public class MyScheduler
{
    public MyScheduler(IJobFactory myJobFactory)
    {
	    _scheduler = new StdSchedulerFactory().GetScheduler().Result;
		_scheduler.JobFactory = myJobFactory;
    }

    public void Start()
    {
        var job = CreateJob<SomeJob>();
        var trigger = TriggerBuilder.Create().StartNow()
            .WithCronSchedule("0/10 ...").Build();
        _scheduler.ScheduleJob(job, trigger);
        _scheduler.Start();
    }

    public IJobDetail CreateJob<T>() where T : IJob
    {
        return JobBuilder.Create<T>()
            .WithIdentity(typeof(T).Name)
            .Build();
    }
}

建立擴充方法

public static class QuartzExtensions 
{
    public static void UseQuartz(this IApplicationBuilder app)
    {
        var scheduler = app.ApplicationServices.GetService<IMyScheduler>();   
        scheduler.Start();
    }
}

在 Startup.cs 檔案中註冊好所有的物件並呼叫 UseQuartz() 方法啟動排程

public void Configure(IApplicationBuilder app, IHostingEnvironment env)
{
    app.UseQuartz();
}

如此一來我們可以透過 CreateJob() 建立 Job 物件, DI 會自動幫我們注入所需的依賴物件.
