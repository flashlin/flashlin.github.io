# IoC 進階應用

# 使用相同的 IService 但在不同的 Controllers 卻又希望使用不同的實際 Service

你問我在dotnet core 能做到嗎? 當然可以啊! 實際上, 有幾種方法可以解決此問題.

首先設計一個泛型 IMyService

public interface IMyService<T> {}

再設計兩個 interface 表明有兩種實體 Service 類型

public interface IDog { }
public interface ICat { }

兩個實際的 Service 都繼承 IMyService, 如下所示

public class DogService : IMyService<IDog> { ... }
public class CatService : IMyService<ICat> { ... }

然後你按照下面方式註冊服務

services.AddScoped<IMyService<IDog>, DogService>();
services.AddScoped<IMyService<ICat>, CatService>();

通過這種方法, 在應用程序中的任何地方, 呼叫端都使用IMyService, 如果該實體是ICat, 將使用 CatService 來獲取它, 但是如果實體是 IDog, 將使用 DogService 來獲取它.

public SomeController(IMyService<ICat> myService)
{ ... }

最後要註冊

services.AddMvc().AddControllersAsServices();

以上是針對 IRepository<TEntity> 服務設計處理, 想像一下, services.AddScoped<IRepository<Customer>, EfRepository<Customer>>(); services.AddScoped<IRepository<Product>, MySqlRepository<Product>>(); 你可以針對實體是 Customer, 就用 EF 去存取. 但如果實體是 Product, 就用 MySql 去存取

當然或者你也能用另外方法做

public class DogService : IMyService { ... }
public class CatService : IMyService { ... }
services.AddScoped<IMyService, DogService>();
services.AddScoped<CatService>();
services.AddScoped<SomeController>(x => new SomeController(x.GetRequiredService<CatService>>()));
